# C-011: Franchise Core Freeze (Intent Map)

## 1. 개요 (Intent)
PRD-011의 핵심 의도는 Core를 "플랫폼 엔진"이자 "프랜차이즈 본점"으로 제도화하는 것이다. 이를 통해 시스템의 근간이 되는 영역을 안정화하고, 확장은 주변부에서 이루어지도록 유도하는 기준선을 선언한다.

## 2. Intent-Rule-Evidence 매핑

| Intent (의도) | Rule (구현 규칙) | Evidence (검증 증거) |
| :--- | :--- | :--- |
| **기준선(Baseline) 확립** | 현재의 Core 구조를 공식 절차를 통해서만 변경 가능한 기준점으로 선언한다. | `B-011` 계약 문서 및 `main` 브랜치 상태 |
| **무질서한 변경 종료** | 모든 Core 수정에 대해 PRD 기반의 공식 절차를 강제한다. | Core 변경 PR과 연결된 PRD 존재 여부 |
| **레이어 간 역할 분리** | Core는 불변 엔진으로, 제품 확장은 외부 레이어로 분리한다. | Core 내부 직접 확장 금지 원칙 준수 |
| **거버넌스 통합** | 기존 테스트 및 승인 규칙(PRD-010)을 Freeze의 필수 조건으로 연결한다. | `B-011` 내 PRD-010 참조 및 준수 기록 |

## 3. Freeze 선언의 원칙

### 3.1 제도적 보호
- Freeze는 기술적 잠금(Locking)이 아닌 제도적 보호(Governance)이다.
- 개발자의 자율성을 억압하는 것이 아니라, 핵심 로직의 신뢰성을 보장하기 위한 최소한의 절차적 장벽이다.

### 3.2 본점과 지점 모델
- **Core (본점):** 시스템의 본질적 규칙을 담으며 엄격히 관리됨.
- **Expansion (지점):** Core를 활용하여 다양한 요구사항을 실현하며 유연하게 운영됨.

## 4. 시나리오별 처리 원칙

### 4.1 Core 직접 수정 시도
- **상황:** PRD 없이 Core 로직을 직접 수정하려는 시도.
- **원칙:** Freeze 선언 위반으로 간주하여 반려하며, PRD 기반 정식 절차 안내.

### 4.2 실험적 기능 추가
- **상황:** Core에 영향을 줄 수 있는 새로운 기능 실험.
- **원칙:** Core 외부 레이어(Adapter 등)에서 우선 진행하거나, LAB PRD를 통해 검증 후 본점 편입 여부 결정.
