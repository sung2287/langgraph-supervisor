# B-011: Franchise Core Freeze (Contract)

## 1. 개요 (Overview)
본 Contract는 `PRD-011: 프랜차이즈 본점 Core Freeze 선언`에 기반하여, 현재의 Core 구조를 시스템의 공식 기준선(Baseline)으로 선언하고 고정하기 위한 계약이다.

## 2. Freeze 대상 및 경계 (Boundary)

### 2.1 물리적 경계
- **대상 경로:** `src/core/*` 하위의 모든 코드 및 설정.
- **대상 브랜치:** `main` 브랜치에 존재하는 해당 계층.

### 2.2 논리적 구성 요소
- Core Error 모델 및 체계
- Core I/O 불변성 및 계약
- Adapter 레이어와의 경계 정의
- PRD-010에 따른 테스트 거버넌스

## 3. Freeze Baseline 선언 (Baseline Declaration)

### 3.1 기준선 확정
- 본 문서 승인 시점의 `src/core/*` 상태를 "프랜차이즈 본점 기준선"으로 확정한다.
- 이후 발생하는 모든 Core 변경은 공식 절차에 따라 수행된다.

### 3.2 비소급 적용 (Non-Retroactive)
- 본 Freeze 선언은 과거의 기술 부채나 기존 코드의 미비점을 즉시 수정할 것을 요구하지 않는다.
- 현재 상태를 역사적 기준점으로 존치한다.

## 4. 변경 통제 계약 (Change Control)

### 4.1 명시적 PRD 의무화
- Freeze 이후 Core 영역의 모든 수정은 반드시 개별 PRD를 생성하여 타당성을 입증해야 한다.
- 임시 수정, 직접 수정, 예외적 우회 경로는 허용되지 않는다.

### 4.2 거버넌스 준수
- 모든 Core 변경은 `PRD-010`의 최소 테스트 존재 규칙 및 승인 흐름을 반드시 준수해야 한다.

## 5. 제약 사항 (Constraints)
- **확장성 보장:** 본 Freeze는 기능 추가를 원천 금지하는 것이 아니라, 무질서한 직접 수정을 금지하고 제도적 절차를 강제하는 것이다.
- **영역 분리:** 신규 기능 및 실험적 확장은 Core 내부가 아닌 Adapter 혹은 상위 레이어에서 수행함을 원칙으로 한다.
